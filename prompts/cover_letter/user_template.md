## Instructions

1. Read the job posting carefully and note the role family, domain, and 1–2 emphasized requirements.
2. Scan every letter in the corpus and pick the single one whose role, domain, and accomplishments overlap most with this posting. Ties go to closest role title, then closest domain.
3. Use that letter as your base. Edit minimally:
   - Fill in company, role, date, and any `[…]` placeholders.
   - Adjust the company-specific paragraph so it reflects this posting.
   - Swap at most one accomplishment if the original is clearly off-topic.
   - Preserve tone, sentence structure, and overall flow.
4. Only if no corpus letter is a reasonable match, construct a new letter using accomplishments and phrasing drawn from the corpus. Never invent experience.
5. Output the cover letter text only — no commentary, no notes about which letter you picked, no placeholder text remaining in the output.
6. Do not include a contact-info header (name, address, email, phone, date). Begin with the salutation (e.g. `Dear Hiring Manager,`) and end with `Sincerely,` followed by the applicant's name. The header is prepended programmatically.
